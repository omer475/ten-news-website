HOW THE ALGORITHM WORKS (V20):
  1. User opens app → API called with NO user_id (cold start)
  2. Server builds candidate pools: top 30 articles PER CATEGORY (subtopic-diverse)
     Categories: Politics, World, Business, Sports, Entertainment, Tech, Science, Health, Finance, Lifestyle
  3. Scores normalized: 0.7 × within-category-rank + 0.3 × global-quality
  4. Entity blocklist: if user skipped 3+ articles with entity X → X is hard-blocked
  5. Global topic cap: no entity can exceed 15% of total pool
  6. Thompson Sampling bandit picks which CATEGORY to draw from (proper Beta distribution)
  7. Within category: articles scored by entity engagement rate × confidence
     entityScore = 0.5 + (engagementRate - 0.5) × confidence
     confidence = 1 - 1/(1+timesShown) → smooth, no hard thresholds
  8. MMR selection with embedding cosine similarity prevents duplicate stories
  9. Entity frequency cap: dominant entities limited to 2-4 per page
  10. Result: 20 articles sent to user
  11. User reactions (engaged/skipped IDs) sent back as URL params on next request

HOW THE TEST SIMULATES USERS:
  - Each persona has: primary interests (90% engage), secondary (45%), curiosity (30%)
  - Also has AVOID list (hard skip) and category preferences
  - Article tags are matched against these lists to determine tier
  - Engagement probability = base_rate × quality_bonus × fatigue_penalty × random_noise
  - Quality bonus: score≥950 → 2.0x, ≥900 → 1.6x, ≥850 → 1.3x
  - CRITICAL: 'irrelevant' articles with score≥950 get 45% engage chance (curiosity)
  - Dwell times: primary=20s, secondary=14s, curiosity=5s, skip=1.5s

RATING SCALE (honest):
  satisfaction = engaged% + glanced% × 0.3
